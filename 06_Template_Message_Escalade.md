# Template — Message d’escalade utilisateur (néophyte)

## Format à copier/coller

Problème (1 à 3 lignes)
[Explique simplement ce qui bloque, sans jargon.]

Option A : [titre court]
[1 à 3 lignes sur la solution]
Pour : [bénéfice principal]
Contre : [limite principale]
Préconisation : [A/B + pourquoi en 1 ligne]

Option B : [titre court]
[1 à 3 lignes sur la solution]
Pour : [bénéfice principal]
Contre : [limite principale]
Préconisation : [A/B + pourquoi en 1 ligne]

Décision attendue
[Ex: « Choisis Option A ou Option B. »]

---

## Exemple

Problème (1 à 3 lignes)
Le test de paiement échoue sur certains cas limites. On peut soit corriger rapidement pour livrer aujourd’hui, soit renforcer le module pour réduire les risques futurs.

Option A : Correctif rapide
On corrige le bug ciblé et on relance les tests essentiels.
Pour : livraison plus rapide.
Contre : risque de revoir ce sujet plus tard.
Préconisation : Option A si priorité = aller vite.

Option B : Renforcement complet
On refait la partie paiement plus proprement avant livraison.
Pour : meilleure stabilité à moyen terme.
Contre : plus de délai maintenant.
Préconisation : Option B si priorité = robustesse.

Décision attendue
Choisis Option A ou Option B.
